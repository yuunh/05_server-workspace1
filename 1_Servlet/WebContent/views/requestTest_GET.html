Get 방식으로 요청 후 응답페이지 보기

특징1. GET 방식으로 요청 시 url의 Header 영역에 데이터를 포함시켜 요청 => 사용자가 입력한 값(데이터)들이 url 노출됨
=> 보안 유지가 불가능함
=> 즉, 로그인이나 회원가입 같은 경우 GET 방식에 부적합하다
특징2. Header 영역은 전송하는 길이에 제한이 있음
=> 방대한 데이터 담았을 경우 초과된 데이터는 절단 돼서 넘어감
=> 즉, 게시판 작성 같은 경우는 GET 방식에 부적합하다
특징3. 장점이라고 한다면 url에 데이터가 노출되어 있기 때문에 즐겨찾기(북마크) 가능
(url 저정해놨다가 재요청)
=> 즉, 검색 기능은 GET 방식에 적합
